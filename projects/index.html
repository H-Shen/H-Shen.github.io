Dynamic-TopK-Ranking-Service

This project is a web service built using Springboot to dynamically calculate and return the top-k most mentioned strings along with their mention counts. The service includes two primary RESTful-API endpoints:

  • update: Receives a string and updates its mention count.

  • query: Receives an integer k and returns the top-k most mentioned strings along with their mention counts. If there is a tie, the most recently updated string will come out first. If there is a tie again, the string with the smaller alphabetical order will come out first.

The project is built with Springboot framework and utilizes Springboot's default caching strategy. The RESTful APIs support high concurrent access. And Mock MVC unit tests for the APIs are included.

K-Way Merger

This project is a toy implementation of a K-way Merger in Go, designed with functionality and efficiency in mind. The application operates in a multi-step process:

  • It begins by accepting a list of input files along with a specified output file.

  • The core functionality involves sorting integers in each input file in ascending order, with the sorted data then being rewritten back to the respective input files.

  • Finally, the application seamlessly merges all integers from these input files. This merging process is efficiently handled using a binary heap algorithm, culminating in the combined result being written to the designated output file.

This implementation not only demonstrates the practical application of sorting and merging algorithms but also showcases the power and flexibility of Go in handling data-intensive tasks.

A Toy Implementation of Different Caches in C++

This project presents the implementation of four cache replacement policies, each meticulously crafted using single-threaded programming in C++. The policies include:

  • First-In-Last-Out (FILO): A unique approach where the most recently added items are the first to be evicted.

  • First-In-First-Out (FIFO): A classic strategy that removes items in the order they were added, ensuring a fair and sequential process.

  • Least Recently Used (LRU): This policy prioritizes user recency, discarding items that haven’t been accessed for the longest time.

  • Least Frequently Used (LFU): Focusing on usage frequency, this method eliminates items with the least number of accesses.

Each policy is meticulously coded in C++, demonstrating not only the power of the language in handling complex algorithms but also providing a practical viewpoint into the efficiency and applicability of these policies in real-world scenarios.

A Toy Implementation of Arbitrary-Precision Fraction in Java

The project includes the implementation of a Java Arbitrary-precision fraction, utilizing the class BigIntegers for numerator and denominator. Fraction is always kept in the lowest terms. Fraction is immutable and guaranteed not to have a null numerator or denominator. The denominator will always be positive (thus sign is carried by the numerator), and the denominator is not allowed to be zero.
